# Contributing

When contributing to this repository, please first discuss the change you wish to make via issue, email, or any other method with the owners of this repository before making a change.

## Pull Request Process

1. Ensure any install or build dependencies are removed before the end of the layer when doing a build.

2. Update the README.md with details of changes to the interface, this includes new environment variables, exposed ports, useful file locations and container parameters.

3. Increase the version numbers in any examples files and the README.md to the new version that this Pull Request would represent. The versioning scheme this project use is [SemVer](http://semver.org/).

## Reporting bugs

In case you find a bug, post the problem as an issue on this repository or ask in the [offical server](https://discord.gg/7bEmxG) and include additional details to help developers reproduce the problem.

## Suggesting a feature

If you want to suggest a new functionality, open an issue explaining the new functionality. Suggestion issues may be closed without any reasons.
